## Шаг 1. Цель MVP и метрика успеха

**Формулировка цели по шаблону:**

> «На MVP мы проверяем, что **компания друзей 18–30 лет** готова **провести полноценную игру в мафию через Telegram-бот без живого ведущего**, потому что получает **голосового ИИ-ведущего с характером, который ведёт игру от первой роли до финального объявления победителя**.
> Успех: **≥70% игр доводятся до конца без блокирующих ошибок**, **≥30% участников тестового плейтеста говорят "заплатил бы 99₽ за следующую игру"**».»

| Параметр | Значение |
|----------|----------|
| **Сегмент** | Компании друзей 18–30 лет, 5–12 человек, частота 1–4 игры в месяц |
| **Ценность (1 фраза)** | Голосовой ИИ-ведущий с характером — все N игроков участвуют, никто не жертвует собой |
| **Целевое действие на MVP** | Завершение первой полноценной игровой сессии через Telegram-бот (от /start до объявления победителя) |
| **Метрика 1** | Completion rate: ≥70% запущенных игр доходят до финала без критических сбоев |
| **Метрика 2** | Готовность платить: ≥30% участников плейтеста отвечают «да» на вопрос «заплатишь 99₽ за следующую игру?» |

## Шаг 2. Основной user flow

**Сценарий:** Компания из 8 человек хочет сыграть в мафию пятничным вечером.

| № | Шаг | Действие | Результат |
|---|-----|----------|-----------|
| 1 | **Триггер** | Кто-то из компании вспоминает про приложение, открывает Telegram | Бот найден по @GameMasterBot |
| 2 | **Вход** | Организатор отправляет `/start` | Приветственное сообщение с голосовым фрагментом ведущего и кнопками «Создать игру» |
| 3 | **Настройка** | Нажимает «Создать игру», выбирает кол-во игроков (8) и характер («Зловещий») | Бот генерирует уникальный код сессии `AX7K2M` и ссылку-приглашение |
| 4 | **Сбор игроков** | Организатор отправляет ссылку в общий чат. Каждый игрок переходит и нажимает `/join AX7K2M` | Бот в общем чате: «Подключились: Саша, Петя, Маша… (8/8)» |
| 5 | **Старт игры** | Организатор нажимает «Начать игру» | Роли автоматически распределяются. Каждый игрок получает **личное голосовое сообщение** с ролью (напр. «Тебе досталась роль Шерифа. Ты — страж города…») |
| 6 | **Ночь 1** | ИИ в общем чате: «Город засыпает. Просыпается Мафия» (голосовое) | Мафия получает личное сообщение с кнопками выбора жертвы. Шериф — с кнопками проверки. Доктор — с кнопками лечения |
| 7 | **Ночные действия** | Каждая спецроль выбирает цель через кнопки в личке | Бот фиксирует все действия, обрабатывает взаимодействия (доктор спасает / мафия убивает) |
| 8 | **Утро** | ИИ в общем чате объявляет итог ночи голосовым сообщением: «Эту ночь пережили все. Мирный город просыпается в недоумении…» | Переход к дневной фазе |
| 9 | **День 1: обсуждение** | Таймер 3 минуты. Игроки обсуждают в чате. ИИ делает провокационные ремарки голосом | Активное обсуждение |
| 10 | **Голосование** | ИИ объявляет голосование. Каждый игрок получает кнопки с именами живых игроков | Бот считает голоса, объявляет выбывшего: «Большинством голосов покидает город Петя» (голосовое) |
| 11 | **Продолжение** | Шаги 6–10 повторяются до условия победы | Игра ведётся автоматически, контекст хранится |
| 12 | **Финал** | ИИ определяет победителя и объявляет финальным голосовым монологом | Итоговое сообщение с результатами, предложение сыграть ещё раз («Сыграть снова»), просьба оценить игру |

## Шаг 3. Альтернативный сценарий

**Тип:** Отключение игрока / потеря соединения в середине игры

| № | Шаг | Описание |
|---|-----|----------|
| 1 | **Триггер** | Игрок «Маша» закрыл Telegram во время ночной фазы. Бот не получил ответ от Маши в течение 45 секунд |
| 2 | **Напоминание** | Бот отправляет Маше личное сообщение: «Все ждут твоего хода. Нажми кнопку ниже, чтобы продолжить» |
| 3 | **Таймаут** | Ещё 30 секунд нет ответа. Бот сообщает в общий чат: «Маша не отвечает. Ждём ещё немного…» |
| 4 | **Решение организатора** | Организатор получает кнопки: «Пропустить ход Маши» / «Ждать ещё 1 минуту» |
| 5 | **Пропуск хода** | Организатор нажимает «Пропустить». Бот записывает в лог: ход Маши пропущен | Игра продолжается |
| 6 | **Возвращение игрока** | Маша возвращается в Telegram, пишет `/rejoin AX7K2M` | Бот восстанавливает сессию по device_id, показывает Маше актуальное состояние игры |
| 7 | **Информирование** | Маша получает сводку: «Ты пропустила Ночь 2. Убит Коля (Мирный). Сейчас идёт День 2. Живы: Саша, Петя, Ты, Вася» | Игрок включается в следующий ход |

## Шаг 5. Модель данных / сущностей MVP

**1. Session (Игровая сессия)**
- Поля: `id (UUID)`, `code (VARCHAR 6)`, `player_count (INT 5–16)`, `status (waiting / active / finished)`, `host_character (zловещий / весёлый)`, `created_at`, `ended_at`
- Связи: содержит многих Players, проходит через GamePhases, логирует GameEvents

**2. Player (Игрок)**
- Поля: `id (UUID)`, `session_id (FK)`, `name (VARCHAR)`, `role_id (FK)`, `status (alive / dead)`, `device_id (VARCHAR)`, `join_order (INT)`
- Связи: принадлежит Session, имеет Role, совершает NightActions, участвует в DayVotes

**3. Role (Роль — справочник)**
- Поля: `id (UUID)`, `name (Мафия / Шериф / Доктор / Мирный)`, `team (mafia / city)`, `abilities (JSON)`, `night_action_type (kill / check / heal / null)`
- Связи: назначается многим Players

**4. GamePhase (Игровая фаза)**
- Поля: `id (UUID)`, `session_id (FK)`, `phase_type (night / day)`, `phase_number (INT)`, `started_at`, `ended_at`
- Связи: принадлежит Session, содержит NightActions и DayVotes

**5. GameEvent (Игровое событие — лог)**
- Поля: `id (UUID)`, `session_id (FK)`, `phase_id (FK, nullable)`, `event_type (player_joined / game_started / player_died / vote_result / game_finished / …)`, `payload (JSON)`, `created_at`
- Связи: привязан к Session и Phase; используется для восстановления состояния при реконнекте

**6. User / Subscription (Пользователь и подписка)**
- `User`: `id`, `device_id (UK)`, `email (nullable)`, `created_at`
- `Subscription`: `id`, `user_id (FK)`, `plan (free / pro)`, `period_start`, `period_end`, `status`
- Связи: User имеет Subscriptions; подписка определяет, доступен ли Pro-режим (до 12 игроков, доп. характеры)

## Шаг 7. Definition of Done для MVP

MVP считается готовым, если выполнены все следующие критерии:

1. **Реализованы все Must-истории** (US-01 — US-09) и прошли ручное тестирование на реальной игровой сессии
2. **Основной user flow проходит без блокирующих ошибок**: игра от `/start` до объявления победителя завершается корректно в ≥90% тестовых запусков
3. **Голосовой движок работает в рамках таймингов**: задержка TTS (Yandex SpeechKit) ≤ 3 секунды в 90% случаев
4. **Обработаны ключевые ошибки**: пустые состояния (никто не подключился), неверный код сессии, таймаут ночного действия, повторное подключение (US-10)
5. **Игровая логика верифицирована**: условие победы (мафия ≥ город / мафия = 0) рассчитывается корректно во всех комбинациях ролей
6. **Сохранность сессии**: контекст игры (убитые игроки, фазы, результаты голосований) сохраняется в БД и не теряется при перезапуске бота
7. **Настроен сбор минимум 2 метрик**: (1) процент завершённых игр, (2) оценка качества голоса по шкале 1–10 через inline-кнопки по итогу сессии
8. **Себестоимость сессии подтверждена**: тестовая игра на 6–8 игроков (~50 реплик ведущего) обходится ≤ 35₽ (TTS + LLM)
9. **Подготовлен сценарий демо**: пошаговый скрипт демо-игры на 6 человек (15–20 минут) с заготовленными триггерными репликами для демонстрации характера ведущего
10. **Задокументировано**: инструкция запуска бота для тестовой группы — 1 страница (как запустить, какие команды, как подключиться)
11. **Зафиксированы ограничения MVP**: список всего, что **не входит** в MVP (веб-интерфейс, D&D-режим, рейтинговая система, кастомные правила, B2B-панель, доп. характеры «Весёлый» / «Детский»)
12. **Проведён минимум 1 тестовый плейтест** с реальными пользователями (≥5 человек) и зафиксированы результаты

## Шаг 8. План демо / пилота

### Кому показываем

5–10 контактов из целевого сегмента:

| # | Профиль | Формат |
|---|---------|--------|
| 1–3 | Студенческие компании из вуза (18–22 года), регулярно играют в мафию | Оффлайн-плейтест |
| 4–6 | Компании друзей 23–28 лет, знакомые команды (корпоративные вечера, тусовки) | Созвон + Telegram-демо |
| 7–8 | Администраторы 2–3 антикафе в Москве | Коммерческое предложение + онлайн-демо |
| 9–10 | Активные участники сообществ по настольным играм ВКонтакте / Telegram | Демо в канале |

### Как показываем

- **Оффлайн-плейтест**: 6–8 человек, 1 игра (~20–30 минут). Без объяснений — просто запускаем и наблюдаем
- **Онлайн-демо**: созвон в Telegram, организатор демонстрирует создание сессии, другие участники подключаются со своих устройств
- **Для антикафе**: PDF + 10-минутный скринкаст с комментарием об экономии на ведущих

### 5 вопросов для пользователей

1. «Что тебе понравилось / не понравилось в процессе игры?» *(открытый, без наводки)*
2. «Голос ведущего создавал атмосферу или мешал?» *(ключевая гипотеза H1)*
3. «Была ли хоть одна момент, когда ты думал "бот облажался" — что именно произошло?»
4. «Заплатил бы 99₽, чтобы сыграть снова в следующий раз? Почему да / нет?» *(гипотеза H2)*
5. «Что одно изменил бы в первую очередь?»

### Что считаем успехом

| Сигнал | Порог |
|--------|-------|
| Completion rate (игры до конца без критических сбоев) | ≥70% |
| Средняя оценка голоса ведущего (1–10, без наводки) | ≥7 |
| Готовность платить 99₽ за следующую игру | ≥30% |
| Предпочтение нашего голоса текстовым ботам (True Mafia / Mafiko) | >70% |
| Упоминание задержки ≥ 2 сек как проблемы | <30% (гипотеза H3) |
