## 1. Уточнённое ценностное предложение

**Операционализированная формулировка:**

> «Для **компаний друзей 18–30 лет**, собирающихся сыграть в Мафию дома, в антикафе или на корпоративе, в ситуации **когда некому быть ведущим или никто не хочет жертвовать своим участием**, приложение обеспечивает **полноценную игру с голосовым ведущим без участия живого человека**, сокращая время на организацию с 10–15 минут до 2–3 минут и устраняя конфликт «кто сегодня жертвует собой», по сравнению с текстовыми приложениями-ведущими (True Mafia, Мафия Ведущий) и живым ведущим, при условии наличия смартфона у каждого участника и Wi-Fi / локальной сети.»

**Основания доверия (почему пользователь поверит):**

1. **Нет задержек, нет интернета.** Все ~300–400 голосовых фраз хранятся в приложении как mp3-файлы, сгенерированные заранее через ElevenLabs / Yandex SpeechKit. Воспроизведение — задержка < 300 мс, не зависит от качества соединения.
2. **Конфиденциальность ролей встроена в механику.** Каждый игрок видит на своём экране только свою роль и свои действия. Мафия не может «засветиться» перед соседом.
3. **Нулевая стоимость входа.** Free-версия (до 5 игроков, без рекламы в самой игре) не требует регистрации и оплаты — снимает барьер «попробовать в первый раз».

## 2. Основной пользовательский сценарий

**«Организатор создаёт игру с нуля»**

| Шаг | Кто | Действие | Что происходит в приложении |
|-----|-----|----------|-----------------------------|
| 1 | Организатор | Открывает приложение → «Создать сессию» | Генерируется уникальный 6-символьный код сессии |
| 2 | Организатор | Выбирает число игроков (5–16) и набор ролей | Приложение рассчитывает состав: 1 мафия на 3 мирных + спецроли |
| 3 | Все игроки | На своих телефонах вводят код и имя | Список участников появляется на экране организатора в реальном времени |
| 4 | Организатор | Нажимает «Начать игру» | Роли раздаются случайно — каждый видит только свою роль |
| 5 | Приложение | — | Ведущий озвучивает приветствие и вступительные правила |
| 6 | Все игроки | Обсуждают вслух (дневная фаза) | Экран показывает фазу «День», опционально — таймер обсуждения |
| 7 | Организатор | Нажимает «Начать голосование» | Каждый голосует на своём экране — один раз |
| 8 | Приложение | — | Ведущий объявляет результат дня (кто выбыл), переход к ночи |
| 9 | Игроки по ролям | Выполняют ночные действия на своих экранах (Мафия — жертва, Шериф — проверка, Доктор — лечение, Мирные — ждут) | Приложение собирает выборы; мирные видят экран ожидания |
| 10 | Приложение | — | Ведущий озвучивает итог ночи, возврат к шагу 6 |
| 11 | Приложение | — | При выполнении условия победы — ведущий объявляет победителя, раскрывает все роли |

## 3. Альтернативный сценарий

**«Повторная игра с теми же участниками»**

| Шаг | Кто | Действие | Что происходит |
|-----|-----|----------|----------------|
| 1 | Приложение | — | После объявления победителя появляется кнопка «Сыграть ещё» |
| 2 | Организатор | Нажимает «Сыграть ещё» | Приложение предлагает: «Те же игроки» / «Изменить состав» |
| 3 | Организатор | Выбирает «Те же игроки» | Все участники остаются подключены — повторный ввод кода не нужен |
| 4 | Организатор | При необходимости корректирует набор ролей | Можно добавить / убрать спецроли |
| 5 | Приложение | — | Роли перераспределяются заново, случайно |
| 6 | Приложение | — | Новая игра стартует с шага 5 основного сценария |

## 4. Карта функций

### Core — ядро MVP (без этого ценность не достигается)

| # | Функция | Шаг сценария | Элемент ценности |
|---|---------|--------------|-----------------|
| C1 | Создание сессии с уникальным 6-символьным кодом | 1–2 | Быстрый старт без живого ведущего |
| C2 | Подключение игроков по коду (каждый на своём телефоне) | 3 | Все участники в одной сессии |
| C3 | Автоматическая случайная раздача ролей (конфиденциально) | 4 | Конфиденциальность и честность |
| C4 | Воспроизведение голосовых фраз по игровым триггерам (mp3, офлайн) | 5, 8, 10, 11 | Атмосфера живого ведущего без задержек |
| C5 | Ночные ролевые экраны с кнопками действий (Мафия / Шериф / Доктор / Мирный) | 9 | Конфиденциальность ролей, удобный ввод |
| C6 | Дневное голосование об исключении игрока | 7 | Полноценный игровой цикл |
| C7 | Отслеживание состояния игры (живые/мёртвые, фаза, счётчик ходов) | Все | Автоматизация судейства |
| C8 | Автоматическое определение и объявление победителя | 11 | Завершение игры без ручного вмешательства |

### Support — повышает качество и удобство

| # | Функция |
|---|---------|
| S1 | 2–3 случайных варианта фразы на каждый игровой триггер (создаёт ощущение «живости») |
| S2 | Экран ожидания с анимацией для мирных игроков в ночную фазу |
| S3 | Таймер обсуждения в дневной фазе (настраиваемый) |
| S4 | Экран правил и описание ролей для новичков |
| S5 | Анимированный персонаж-ведущий на экране (фирменный маскот) |
| S6 | Режим повторной игры (rematch) без повторного ввода кода |
| S7 | Настройки звука: громкость, скорость речи |
| S8 | Раскрытие всех ролей в финальном экране |

## 5. Backlog ядра MVP

| ID | User Story | Acceptance Criteria | Priority | Риски / Зависимости |
|----|-----------|---------------------|----------|---------------------|
| US-01 | Как **организатор**, я хочу создать игровую сессию, чтобы друзья могли подключиться одним кодом без лишних действий | 1. Сессия создаётся за < 3 сек. 2. Генерируется уникальный 6-символьный код. 3. Список подключившихся обновляется в реальном времени. 4. Сессия активна до завершения игры или ручного закрытия | Must | — |
| US-02 | Как **игрок**, я хочу подключиться к сессии по коду, чтобы участвовать в игре на своём телефоне | 1. Подключение за < 5 сек. 2. При неверном коде — понятное сообщение об ошибке. 3. Подключение доступно только до старта игры. 4. Игрок видит имена уже подключившихся | Must | Зависит от US-01 |
| US-03 | Как **организатор**, я хочу запустить игру, чтобы роли раздались автоматически и случайно | 1. Минимум игроков для старта — 5. 2. Каждый получает роль случайно, видимую только ему. 3. Соотношение: 1 мафия на 3 мирных. 4. После старта добавить игрока нельзя | Must | Зависит от US-02 |
| US-04 | Как **игрок (Мафия)**, я хочу видеть список мирных ночью и выбрать жертву кнопкой, чтобы моё действие было конфиденциальным | 1. Показываются только живые мирные игроки. 2. Выбор фиксируется без голосового ввода. 3. При ≥ 2 мафиях — жертва определяется большинством голосов. 4. Экран мафии не виден другим | Must | Зависит от US-03 |
| US-05 | Как **игрок (Шериф)**, я хочу проверить одного участника ночью, чтобы узнать его принадлежность к команде | 1. Результат виден только Шерифу. 2. Проверка доступна только в ночную фазу. 3. Нельзя проверить мёртвого игрока. 4. Предупреждение при повторной проверке того же игрока | Must | Зависит от US-03 |
| US-06 | Как **игрок (Доктор)**, я хочу вылечить одного участника ночью, чтобы защитить его от мафии | 1. Нельзя вылечить одного и того же два раза подряд. 2. Нельзя вылечить мёртвого. 3. Если мафия выбрала ту же жертву — игрок выживает. 4. Результат не раскрывается другим | Must | Зависит от US-03 |
| US-07 | Как **все игроки**, я хочу слышать голосового ведущего при смене фаз, чтобы следить за игрой без чтения текста | 1. Звук воспроизводится с задержкой < 300 мс после триггера. 2. На каждый триггер — один из 2–3 случайных вариантов фразы. 3. Ведущий корректно называет имена выбывших. 4. Громкость регулируется на устройстве | Must | Требует готовой аудиобиблиотеки (~300–400 mp3) |
| US-08 | Как **организатор**, я хочу провести дневное голосование, чтобы игроки выдвинули и исключили подозреваемого | 1. Каждый живой игрок голосует ровно один раз. 2. При ничье — повторное голосование. 3. Результат озвучивает ведущий. 4. Выбывший переходит в статус «мёртв» | Must | Зависит от US-07 |
| US-09 | Как **приложение**, я хочу автоматически определять победителя, чтобы игра завершалась без ручного вмешательства | 1. Победа мафии — мафия ≥ мирных (живых). 2. Победа города — все мафии устранены. 3. Ведущий озвучивает победителя и раскрывает роли. 4. После победы доступна кнопка «Сыграть ещё» | Must | Зависит от US-07, US-08 |
| US-10 | Как **мирный игрок**, я хочу видеть экран ожидания ночью, чтобы не мешать ночным ролям и не выдать себя пассивностью | 1. Мирный видит текст / анимацию «Город спит». 2. Кнопки действий недоступны. 3. Переход к дню происходит автоматически после всех ночных действий. 4. Визуально экран не отличается от экрана спецроли в ожидании | Should | Зависит от US-03 |

## 6. Нефункциональные требования

| # | Категория | Требование | Обоснование |
|---|-----------|-----------|-------------|
| NFR-01 | Производительность | Воспроизведение аудиофразы — задержка < 300 мс после игрового триггера | Задержка > 1 сек разрушает атмосферу живого ведущего |
| NFR-02 | Производительность | Синхронизация состояния игры между устройствами — < 1 сек в условиях локальной Wi-Fi сети | Рассинхрон фаз и голосований дезориентирует игроков |
| NFR-03 | Надёжность | При разрыве соединения приложение восстанавливает роль и текущую фазу за < 10 сек после переподключения | Разрыв у одного игрока не должен прерывать игру остальных |
| NFR-04 | Автономность | Все аудиофайлы (mp3) хранятся локально; воспроизведение не требует интернет-соединения | Нестабильный интернет в антикафе и дома; полное устранение зависимости от API-провайдеров в рантайме |
| NFR-05 | Безопасность | Роль игрока не передаётся на другие устройства; код сессии действует только в рамках одной игры | Случайное раскрытие роли необратимо ломает игру |
| NFR-06 | Совместимость | Поддержка iOS 15+ и Android 10+; корректный UI на экранах от 4.7 дюймов | Охватывает > 90% актуальных устройств целевой аудитории 18–30 лет |
| NFR-07 | Масштабируемость | До 16 игроков в одной сессии без деградации производительности (Free — до 5) | Граница Free/Pro — ключевой триггер конверсии в подписку |
| NFR-08 | Доступность интерфейса | Все интерактивные элементы — минимум 44 × 44 pt (Apple HIG / Material Design); поддержка системного размера шрифта | Ночная фаза: пользователь смотрит в телефон при слабом освещении, нажатие должно быть точным |
| NFR-09 | Хранение данных | Free-версия не собирает персональные данные; история партий хранится локально на устройстве | Снижает барьер доверия при первом запуске; соответствие 152-ФЗ без необходимости авторизации |
| NFR-10 | Архитектура / Интеграции | Отсутствие зависимости от внешних API в рантайме; TTS используется только на этапе разработки (однократная генерация mp3) | Исключает операционные риски: смена провайдера, рост стоимости API, падение сервиса во время сессии |
